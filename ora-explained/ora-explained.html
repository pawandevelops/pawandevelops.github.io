INSERT statements in Oracle SQL

INSERT statement or command is a DML command or Data Manipulation Language command. DMLs affect the data stored in a table, and don't affect the structure of a table. An Insert statement helps to add a row to a table ( or in some cases, a view ).

Commands that modify data are called DMLs. DMLs involve inserting data into tables, retrieving existing data, deleting data from existing tables and modifying existing data.

Other than INSERT; UPDATE & DELETE are also DMLs. The names of these commands also hint at the obvious functions which they may perform in a table (generally), i.e. UPDATE is used to update a record and DELETE is used to remove a record.

Before we go ahead and execute an INSERT statement, we must have a table to add records. In the following DDL (Data Definition Language) i.e. CREATE TABLE, we have created the table Acctmanager.

The table Acctmanager has columns: amId, amFirst, amLast, amDate, amSal, amComm and region. The table has amId as its primary key. This means all amIds must be unique and they cannot a NULL value. A CHECK constraint has also been placed on the table. This constraint allows only eight accepted values that could be entered for the Region column.

The following CREATE TABLE statement creates the Table Acctmanager in the database: 

SQL> CREATE TABLE acctmanager
  2  (amid CHAR(4),
  3  amfirst VARCHAR2(12) NOT NULL,
  4  amlast VARCHAR2(12) NOT NULL,
  5  amedate DATE DEFAULT SYSDATE,
  6  amsal NUMBER(8,2),
  7  amcomm NUMBER(7,2) DEFAULT 0,
  8  region CHAR(2),
  9  CONSTRAINT acctmanager2_amid_pk PRIMARY KEY (amid),
 10  CONSTRAINT acctmanager2_region_ck
 11  CHECK (region IN ('N', 'NW', 'NE', 'S', 'SE', 'SW', 'W', 'E')));

Table created.

Once the table Acctmanager has been created, we can start inserting records in it by using INSERT statements.

The INSERT statement has the following syntax:

INSERT INTO tablename [(column name1, column name2...)] VALUES ( data value1, data value2 ... );

Here, table name is the table in which we want to insert a row into. In our case, we are going to insert a row into the table Acctmanager.

In the above INSERT statement syntax, column names are optional. This means they may be omitted in certain cases as per our need.

The keywords INSERT, INTO and VALUES must be written exactly as they are. Although Oracle SQL is not case sensitive, but by convention we will use these keywords in uppercase.

First Case: Without a column list

This is done when all input column values for a row are given. If the data entered in the VALUES clause contains a value for every column and is in the same order as columns in the table, column names can be omitted in the INSERT INTO clause.

The INSERT statement for the first case:

SQL> INSERT INTO acctmanager
  2  VALUES ( 'T500', 'NICK', 'TAYLOR', '05-SEP-2009', 42000, 3500, 'NE');
       
1 row created.

We can check the newly inserted row in the table Acctmanager with the help of following SQL statement: 

SQL> SELECT * FROM Acctmanager;

AMID AMFIRST      AMLAST       AMEDATE        AMSAL     AMCOMM REGION
---- ------------ ------------ --------- ---------- ---------- ----------
T500 NICK         TAYLOR       05-SEP-09      42000       3500 NE

Second Case: When one or more columns is omitted

If you enter data for only some columns, or if columns are listed in a different order than they’re listed in the table, the column names must be provided in the INSERT INTO clause in the same order as they’re given in the VALUES clause. You must list the column names inside parentheses after the table name in the INSERT INTO clause.

  • If more than one column is listed, column names must be separated by commas.
  • If more than one data value is entered, the values must be separated by commas.
  • You must use single quotes to enclose nonnumeric data inserted in a column.

The INSERT statement for this case is as follows:

SQL> INSERT INTO acctmanager (amid, amfirst, amlast, amsal, amcomm, region)
  2  VALUES('J500', 'SAMMIE', 'JONES', 39500, 2000, 'NW');
  
1 row created.

In the above INSERT statement, the column amedate wasn't listed in the column list. There are only six column names mentioned in the column list. The VALUES clause of the statement contains the corresponding values for the columns.

The amedate column's value would be inserted by default as the system date. This is because in the CREATE TABLE we declared amedate to be of DATE datatype and to have a DEFAULT SYSDATE value in case it is omitted in the INSERT statement.

To see the record having amid as J500, we use this SELECT query, which is as follows: 

SQL> SELECT * FROM acctmanager WHERE amid = 'J500';

AMID AMFIRST      AMLAST       AMEDATE        AMSAL     AMCOMM REGION
---- ------------ ------------ --------- ---------- ---------- ----------
J500 SAMMIE       JONES        02-OCT-21      39500       2000 NW

In the above SELECT query, we can see that column amedate has the date: 02-OCT-21 as its value. This is the date of the system on which ORACLE database was hosted, i.e. it is the system date. Hence, a column name can be omitted and a record could be added to a table in this fashion also.

Third Case: Inserting a NULL value

In the above case, suppose we are not sure to include amedate of a record, then we can simply use the keyword NULL. Also, like our fist case, here the column list could be omitted, but then the VALUES list must contain corresponding column values in the same order as the columns were declared in the CREATE TABLE statement.

The INSERT statement for this case is as follows:

SQL> INSERT INTO acctmanager
  2  VALUES ('L500', 'MANDY', 'LOPEZ', NULL, 47000, 1500, 'NW');
               
1 row created.

This INSERT statement would add a record for the person Mandy Lopez with amedate as NULL. We can check the latest inserted row with the help of the following SELECT query: 

SQL> SELECT * FROM acctmanager WHERE amid = 'L500';

AMID AMFIRST      AMLAST       AMEDATE        AMSAL     AMCOMM REGION
---- ------------ ------------ --------- ---------- ---------- ----------
L500 MANDY        LOPEZ                       47000       1500 NW

We can see the output of the SELECT query. Here under the column name Amedate, the correspoding record value is NULL, which is represented as a blank here.

Once we have finished our work of adding records to a table, a COMMIT command must be issued in order to save the work. This command is irreversible and helps in making the record permanent in the table. This could be done as follows:

SQL> commit;

Commit complete.

So far, we have inserted three records to the table. In order to view them all, we would use the following SELECT query: 

SQL> SELECT * FROM acctmanager;

AMID AMFIRST      AMLAST       AMEDATE        AMSAL     AMCOMM REGION
---- ------------ ------------ --------- ---------- ---------- ----------
T500 NICK         TAYLOR       05-SEP-09      42000       3500 NE
J500 SAMMIE       JONES        02-OCT-21      39500       2000 NW
L500 MANDY        LOPEZ                       47000       1500 NW

This was all about the INSERT statement or command in Oracle SQL.
